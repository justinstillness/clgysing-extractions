1. LIBFORGE OVERVIEW & THREE WORKFLOWS
================================================================================

LibForge is the content production framework that transforms raw research
materials into publication-ready content through three interconnected
workflows:

WORKFLOW 1: BOTTOM-UP EXTRACTION
  Raw source materials -> organized, extracted, atomic concept data
  "Mining the ore"

WORKFLOW 2: TOP-DOWN ARCHITECTURE
  Master thesis -> thesis tree -> series organization
  "Designing the blueprint"

WORKFLOW 3: THE HANDSHAKE & DRAFTING
  Bottom-up data + top-down structure -> argument blocks -> prose
  "Building the house"

These workflows are not strictly sequential. Bottom-up extraction can
proceed independently. Top-down architecture can begin with raw notebooks
(validated by preliminary testing). However, the Handshake requires both
workflows to have produced their outputs.

The three workflows contain the following stages:

  BOTTOM-UP (Workflow 1):
    Stage BU-1: Source Package Preparation
    Stage BU-2: Pre-Extraction Evaluation (PE)
    Stage BU-3: Thematic Mapping (TM)
    Stage BU-4: Core Extraction (TX, VX, TS)
    Stage BU-5: Gap Detection & Fill (GD, GX)
    Stage BU-6: Evaluation & Classification (EV, CL)
    Stage BU-7: Glossary Generation (GL)
    Stage BU-8: D1 Library Assembly

  TOP-DOWN (Workflow 2):
    Stage TD-1: Thesis Genesis (Master Thesis Formation)
    Stage TD-2: Thesis Tree Development (Epistemic Regress)
    Stage TD-3: Series Organization
    Stage TD-4: Thesis-Lens Extraction (TL)

  HANDSHAKE & DRAFTING (Workflow 3):
    Stage HS-1: Argument Block Creation (ArgBlocks)
    Stage HS-2: Editorial & Style Preparation
    Stage HS-3: Drafting
    Stage HS-4: Refinement & Publication
