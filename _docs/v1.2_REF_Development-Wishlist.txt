W-063: Repository Inventory & Navigation Document
--------------------------------------------------------------------------------
Priority:    P3
Status:      IDEA (S027)

Description:
Create and maintain a living inventory document for the clgysing-extractions
GitHub repository. As file count grows (400+ files), navigating the repo
from the File Browser or GitHub UI becomes challenging.

The inventory should:
- List all directories with purpose descriptions
- Catalog key files in _docs/ with version and description
- Summarize extraction file counts by project/phase/pass-type
- Be auto-generatable from Indexer-compliant filenames (no content reading)
- Serve as a quick-reference "table of contents" for the repo

Integration opportunity: If Indexer naming conventions (W-050) are fully
adopted, the inventory can be generated programmatically by parsing
filenames — merging navigation and naming standardization into one system.
Caliper recursive scan (W-049) could feed the data; Chronicler writes
the output.

Approaches:
- Manual: Claude generates via Chronicler:list_files (recursive) + write
- Semi-auto: Script parses Indexer-compliant filenames into structured doc
- Full auto: Caliper enhancement generates inventory as part of analysis

Could live at: repo root (REPO-INVENTORY.txt) or _docs/

Relates to: W-049 (Caliper recursion), W-032 (Metadata Index), W-050 (Indexer)

Build estimate: <1 session (manual), 1 session (scripted)
