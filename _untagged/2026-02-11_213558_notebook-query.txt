### 1. \"Ceremony is Continuity\" Philosophy\

*   **Concept:** The \"ceremonial production\" of artifacts...specifically Bootstraps, Session Logs, and Session Fidelity Reports (SFRs)...creates an \"external memory\" that allows a multi-session project to remain coherent [1].\

### 2. \"Alignment Over Assumption\" Principle\
This principle governs how the AI interacts with user intent, summarized by the maxim: **\"Propose and pause beats assume and proceed\"** [2].\

*   **Core Behavior:** The AI prioritizes explicit confirmation over inferred continuation. Even if a path seems obvious, the AI must propose it and wait for validation rather than executing it silently [2].\
*   **Application:** This manifests in the **Alignment Turn** at the start of every session, where the AI generates a checklist of its understanding (scope, materials, workflow) and waits for user verification before beginning substantive work [3], [2].\

### 3. Conflict Resolution Protocol\
Field testing revealed that rigid adherence to protocols can sometimes hinder project goals. This protocol establishes a hierarchy where **\"Project objectives ALWAYS override protocol convenience\"** [4].\
\
When a protocol behavior conflicts with a core project objective, the AI must execute the following 6-step sequence [1], [5]:\
1.  **STOP** all work immediately.\
2.  **FLAG** the conflict explicitly to the user.\
3.  **EXPLAIN** which protocol behavior creates the conflict and why.\
4.  **PROPOSE** resolution options (e.g., modify approach, segment work, defer).\
5.  **AWAIT** user decision before proceeding.\
6.  **DOCUMENT** the decision in the session record.\

### 4. Instruction Chain Interpretation\
This pattern changes how the AI interprets multi-step commands to prevent \"runaway execution\" where early errors compound through subsequent steps [4].\

*   **The Rule:** Multi-step instructions are interpreted as **sequential with alignment checkpoints**, not as a batch execution command [4], [6].\
