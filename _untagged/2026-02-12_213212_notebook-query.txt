# TPL-001: PROJECT INITIATION FORM\
\
**Template ID:** GOV-AIPM-TPL-001\
**Version:** 2.0\
**Purpose:** Structures the rooting process (Phase 0) for new AI-assisted projects. Captures project parameters and establishes shared expectations. Serves as the foundation for Project Guide and Phase Protocol creation.\
**When to Use:** At the start of any new AI-assisted project or major workstream, as part of the rooting (Phase 0) process.\
**Cross-reference:** Section 2.5.1 (Rooting / Phase 0), Section 5.1 (Project Guide), Section 5.2 (Phase Protocol)\
\
---\
\
### INSTRUCTIONS\
\
This form structures the rooting process for a new project. Rooting (Phase 0) is the deliberate process of establishing a project's foundation before execution begins. It produces two foundational outputs: a Project Guide and a Base Phase Protocol.\
\
**HOW TO USE THIS FORM:**\
\
**Option A ... Pre-Completion:** Complete as much of this form as you can before your first AI session. Provide the partially or fully completed form along with this manual at session start. The AI will review your inputs and engage in collaborative dialog to fill gaps, probe assumptions, and refine the project definition.\
\
**Option B .< 2026/02/12 21:32:55.000545515  length=8192 from=16885 to=25076
.. Collaborative Dialog:** Provide this manual and any relevant materials at session start. Tell the AI: \"I'd like to root a new project\" or \"Help me complete TPL-001.\" The AI will walk through each section collaboratively, asking targeted questions to build the project definition together.\
\
Either option leads to the same outcome: a verified Project Guide and Base Phase Protocol. Option B is recommended for complex projects where assumptions need testing through dialog.\
\
**IMPORTANT:** Rooting is design work, not form completion. Expect 10-20 substantive exchanges where the AI probes your thinking, surfaces gaps, and proposes alternatives. This investment in foundation-building pays dividends throughout the project. If rooting takes multiple sessions, that is normal for complex projects.\
\
**Tips:**\
*   Be specific about scope boundaries ... what's OUT is as important as what's IN\
*   Define the destination before mapping the journey ... what does \"done\" look like?\
*   Specify deliverable formats so the AI knows what to produce\
*   Include any constraints the AI should know about\
*   Think about phase structure ... how does the work naturally divide?\
\
See Section 3.5 for the Dialog Intake System.\
