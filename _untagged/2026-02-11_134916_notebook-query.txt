### 1. Response Indexing System Overview\
The Response Indexing System is a continuous, project-wide numbering protocol used to track significant AI outputs (\"turns\") across multiple sessions. Its primary purpose is to maintain a \"Dynamic State\" of continuity, allowing precise referencing of past actions, decisions, and artifacts without needing to re-read entire transcripts.\
\
*   **Continuity**: Indices are sequential across the entire project lifecycle. If Session 001 ends at `[1]`, Session 002 begins at `[2]`.\
*   **Function**: It tags major conversational turns, artifact generation, and protocol acknowledgments.\
*   **Tracking**: The specific range used in a session is recorded in the **Session Log** and the starting index for the *next* session is encoded in the **Bootstrap**.\
\
### 2. Format & Syntax\
The standard format for a response index header is:\
\
`[####] | Semantic Title`\
\
*   **[####]**: A four-digit, zero-padded integer (e.g., `[2]`).\
*   **Semantic Title**: A brief, descriptive title summarizing the action or content of that turn (e.g., `Session 002 Initiation & Protocol Confirmation`).\
\

### 4. Session Initiation & Alignment Turn Protocol\
The indexing system is most critical during the **Session Initiation** phase to establish the \"handshake\" between the user and the AI.\
\
**The Protocol Sequence:**\
\
1.  **User Input (Bootstrap)**: The user provides the `Bootstrap` file (which contains the \"Next Session Starts At Index: [XXXX]\" directive) and the previous `Session Log`.\
2.  **Mandatory First Action (AI)**: The AI reviews the **Phase Protocol** and confirms adherence. This is often combined with the first indexed response.\
    *   *Example:* \"Phase Protocol confirmed... [2] | Session 002 Initiation & Protocol Confirmation\" [3]\
3.  **Context Review & Indexing**: The AI acknowledges the bootstrap and sets its internal counter to the specified index.\
4.  **Alignment Checklist (AI)**: The AI presents a structured checklist to confirm scope, materials, and workflow before starting work.\
    *   *Example:* \"ALIGNMENT CHECKLIST ... Session 002... Confirm alignment?\" [10], [11]\
5.  **User Confirmation**: The user responds (e.g., \"Aligned\" or \"Proceed\"), locking the session scope.\
    *   *Example:* \"Aligned.\" [11]\
